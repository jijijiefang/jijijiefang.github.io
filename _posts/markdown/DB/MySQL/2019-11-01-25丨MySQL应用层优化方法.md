# MySQL应用层优化方法
## 使用连接池
原理:
- 当进程启动时，创建相应的数据库连接池对象；
- 果程序需要请求数据库，则直接从连接池获取到一个连接；
- 数据库请求完成后，释放数据库连接池。
>在《高性能 MySQL》第 14 章：应用层优化中就提到：当遇到连接池完全占满时，应该将连接请求进行排队，而不是扩展连接池。这样可以避免将压力都转到 MySQL 上而导致 MySQL 连接数过多。
